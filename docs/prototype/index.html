Pipeline architecture.

Six layers, strict sequencing. Source files arrive, the processing layer converts them into structured text, Notion carries the relationships, AI fills the gaps, and QC closes the loop.

01 Source
OneDrive and Outlook feed the system.

PDFs, DOCX, XLSX, and CSV email exports enter as raw evidence rather than polished product data.

Inputs: OneDrive, Outlook CSV exports
02 Ingestion
Documents become structured text.

ingest_onedrive.py and parse_emails.ps1 walk folders, match matters, and write usable derivatives.

Scripts: ingest_onedrive.py, parse_emails.ps1, ingest_onedrive.ps1
03 Processing
Dupes disappear and formats settle down.

email_consolidator.mjs, email_to_md.py, docx_to_md.py, and email_defuzz.mjs build a clean working corpus.

Scripts: email_consolidator.mjs, email_to_md.py, docx_to_md.py, email_defuzz.mjs, transcribe_repo_pdfs.py
04 Sync
Notion becomes the case registry.

notion_wire_cases.py, notion_wire_batch.py, and notion_case_dates.py keep matters, dates, and relations aligned.

Scripts: notion_wire_cases.py, notion_wire_batch.py, notion_case_dates.py, notion_consolidate_emails.py, notion_fast_consolidate.py
05 AI Tagging
Models label the content and fill the gaps.

email_deep_tag.mjs, legal_matters_fill.mjs, and legal_hold_backfill.mjs turn raw material into decision-ready structure.

Scripts: email_deep_tag.mjs, email_audit.mjs, legal_matters_fill.mjs, legal_hold_backfill.mjs, legal_matters_pdf.py
06 QC
Every push gets checked against the corpus.

qc_sweep.mjs, repo_sweep.py, verify_repo_consistency.py, and tactical_brief.py keep the front door honest.

Scripts: qc_sweep.mjs, repo_sweep.py, verify_repo_consistency.py, tactical_brief.py, repo_data_push.mjs

AI skill modules.

Six purpose-built Claude skills operate under strict execution boundaries. Each skill has a defined trigger, authority scope, and output format.

Intelligence

Stonewall Brain

Document classification, case timeline construction, email corpus analysis, matter synthesis, and gap analysis with source traceability.

Lookup

Case Lookup

Fast case fact retrieval — hold status, OC contact, trial dates, offer history. Parallel searches across Legal Matters, All Email, and OneDrive.

Search

Case File Finder

Locate case files across OneDrive workspace by client name, matter keywords, date range, and document type with ranked results.

Finance

Billing Reconciler

Reconcile timesheets, prebills, and billing imports. Validates duplicates, missing matter numbers, and zero-hour entries.

Narrative

Legal Chapter

Long-form legal narrative generation with incremental file saves, section-by-section assembly, and citation-backed final output.

Upload

Notion Upload

Bulk CSV/PDF to Notion database uploads with batching, schema validation, and parallel agent execution for large datasets.

System integrations.

The platform connects to three primary external systems. All credentials and database IDs are loaded from environment variables — never hardcoded.

Notion

Legal Matters Case registry with dates, holds, and posture tracking
All Email Email corpus with case relations and AI tags
Document Archive Ingested document derivatives and metadata

OneDrive

Personal Root Personal OneDrive for draft and working files
Firm Root Firm OneDrive for case documents and evidence
Supported Types PDF, DOCX, XLSX, CSV, EML, MSG, TXT, HTML, XML, ZIP

AI Services

Claude API Anthropic Claude for document classification and analysis
OpenAI API GPT models for property fill and tagging
Batch API Anthropic Batch API for high-volume processing jobs

CI/CD workflows.

Four GitHub Actions workflows govern verification, deployment, and code review automation.

verify.yml Runs Python and Node test suites on every push and pull request. Generates consistency reports.
static.yml Deploys the docs/ folder to GitHub Pages on push to main branch.
claude.yml Claude Code agent automation triggered by issue comments, PR reviews, and @claude mentions.
claude-code-review.yml Automated code review powered by Claude Code for pull request analysis.
