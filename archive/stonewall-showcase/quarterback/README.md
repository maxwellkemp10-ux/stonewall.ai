# Quarterback Overview

Stonewall is a legal document intelligence platform built as a litigation control plane. It turns a large working corpus into something counsel can search, validate, synchronize, and operate against in real time using flat files, explicit indexing, AI recall architecture, and static deployment surfaces.

The core claim is simple: legal operations do not need to begin with a heavyweight proprietary platform. They can begin with a rigorously structured corpus. Once every artifact is cataloged with durable IDs, dates, types, matter links, entity references, pattern references, and summary metadata, the same archive can serve multiple jobs at once:

* search layer
* validation layer
* operator layer
* publishing layer
* AI retrieval layer

That is why Stonewall feels coherent. The same source of truth drives the CLI, the portal, the official brief, GitBook, and the AI recall system.

## Innovation stack

### Flat-file searchable database

`catalog/manifest.md` functions as a working database in Markdown. It remains human-readable, Git-trackable, grep-queryable, and structurally inspectable while still carrying the metadata needed for serious operations.

### CLI intelligence layer

The stdlib-only CLI exposes stats, search, matter views, timelines, patterns, and validation from one portable surface.

### AI recall architecture

The brain is routed recall, not opaque memory. Codex files direct the model to source surfaces first, then require fresh reading before assertion.

### Ingestion and synchronization

Documents move from the source reservoir into searchable derivatives, normalized metadata, and operator-facing sync surfaces.

### Workflow leverage

The commercial story is operational: daily dossier generation, deadline intelligence, report-packet readiness, and live deposition outline tailoring.
